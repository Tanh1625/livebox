📋 User Stories – LiveBox Platform

Tài liệu User Stories theo chuẩn Agile (BDD Acceptance Criteria) cho dự án nền tảng chat thời gian thực LiveBox. Phiên bản v0.2 – Sprint Planning Backlog.

🔐 Epic 1 – Authentication & Onboarding
LB-101 Đăng ký tài khoản mới
As a người dùng mới (Lan Anh),
I want to đăng ký tài khoản bằng email và mật khẩu trong một bước duy nhất,
So that tôi có thể bắt đầu sử dụng nền tảng ngay mà không bị cản trở bởi quy trình đăng ký phức tạp.
Acceptance Criteria (BDD)
AC1: Đăng ký thành công
GivenTôi truy cập trang /register và nhập email hợp lệ + mật khẩu ≥ 8 ký tự
WhenTôi nhấn nút "Tạo tài khoản"
ThenHệ thống tạo tài khoản, cấp JWT token và chuyển hướng tôi đến trang Home trong vòng 2 giây
AC2: Email đã tồn tại
GivenEmail tôi nhập đã được đăng ký trong hệ thống
WhenTôi submit form đăng ký
ThenHệ thống hiển thị lỗi inline: "Email này đã được sử dụng. Vui lòng đăng nhập hoặc dùng email khác."
AC3: Mật khẩu không đạt yêu cầu
GivenTôi nhập mật khẩu có ít hơn 8 ký tự
WhenTôi rời khỏi input field mật khẩu (onBlur)
ThenField mật khẩu hiển thị viền đỏ và gợi ý: "Mật khẩu cần ít nhất 8 ký tự."
Priority: High Persona: Lan Anh – New User Tech: Spring Security + JWT
LB-102 Đăng nhập và duy trì phiên làm việc
As a người dùng đã có tài khoản,
I want to đăng nhập bằng email / mật khẩu và duy trì trạng thái đăng nhập giữa các lần truy cập,
So that tôi không phải đăng nhập lại mỗi khi mở ứng dụng.
Acceptance Criteria (BDD)
AC1: Đăng nhập thành công
GivenTôi nhập đúng email và mật khẩu
WhenTôi nhấn "Đăng nhập"
ThenHệ thống trả về Access Token (1h) + Refresh Token (7 ngày), lưu vào localStorage, chuyển hướng đến trang chủ
AC2: Sai thông tin đăng nhập
GivenTôi nhập sai mật khẩu
WhenTôi nhấn "Đăng nhập"
ThenHệ thống hiển thị: "Email hoặc mật khẩu không chính xác." – không tiết lộ trường nào sai
AC3: Token hết hạn – tự gia hạn
GivenAccess Token đã hết hạn nhưng Refresh Token còn hiệu lực
WhenClient gửi request đến bất kỳ API nào
ThenHệ thống tự động gia hạn Access Token mới, request ban đầu hoàn thành bình thường, user không bị đăng xuất
Priority: High Persona: All Users Tech: JWT Refresh Token
LB-103 Đăng xuất an toàn
As a người dùng đã đăng nhập,
I want to đăng xuất khỏi hệ thống và vô hiệu hóa phiên làm việc hiện tại,
So that không ai có thể truy cập tài khoản của tôi trên thiết bị này sau khi tôi rời đi.
Acceptance Criteria (BDD)
AC1: Đăng xuất thành công
GivenTôi đã đăng nhập và nhấn vào avatar → menu dropdown
WhenTôi nhấn "Đăng xuất"
ThenHệ thống xóa Access Token + Refresh Token khỏi localStorage, revoke Refresh Token trên server, đóng kết nối WebSocket, chuyển hướng về trang Login
AC2: Token bị revoke không thể tái sử dụng
GivenTôi đã đăng xuất
WhenMột request sử dụng Refresh Token cũ gọi đến API /auth/refresh
ThenServer trả về HTTP 401 Unauthorized, không cấp token mới
Priority: High Persona: All Users Tech: JWT revoke + WebSocket disconnect
🏠 Epic 2 – Server Management
LB-201 Tạo server mới
As a Server Owner (Anh Đạt),
I want to tạo một server mới với tên và ảnh đại diện tùy chỉnh,
So that tôi có không gian riêng để tổ chức cộng đồng hoặc nhóm làm việc của mình.
Acceptance Criteria (BDD)
AC1: Tạo server thành công
GivenTôi đã đăng nhập và nhấn "Tạo Server"
WhenTôi nhập tên server (≤ 100 ký tự) và nhấn "Tạo"
ThenServer được tạo với kênh mặc định #general, tôi được gán role Owner, server xuất hiện trong sidebar trái
AC2: Upload ảnh đại diện server
GivenTôi đang ở màn hình tạo server
WhenTôi upload ảnh PNG/JPG ≤ 2MB
ThenẢnh được hiển thị preview trước khi lưu; sau khi lưu xuất hiện trong icon server tại sidebar
AC3: Tên server bị bỏ trống
GivenTôi để trống trường "Tên server"
WhenTôi nhấn "Tạo"
ThenNút Tạo bị disable và hiện tooltip: "Tên server không được để trống."
Priority: High Persona: Anh Đạt – Server Owner Tech: REST API + PostgreSQL
LB-202 Mời thành viên qua invite link
As a Server Owner (Anh Đạt),
I want to tạo và chia sẻ invite link để mời người khác vào server,
So that thành viên mới có thể tham gia nhanh mà không cần tôi thêm thủ công từng người.
Acceptance Criteria (BDD)
AC1: Tạo invite link
GivenTôi là Owner của server và mở Settings → Invite
WhenTôi nhấn "Tạo link mời"
ThenHệ thống sinh URL dạng /invite/{code} với thời hạn 7 ngày, hiển thị và có nút "Copy"
AC2: Tham gia server qua invite link
GivenTôi (người dùng đã đăng nhập) truy cập URL invite hợp lệ
WhenTôi nhấn "Tham gia Server"
ThenTôi được thêm vào server với role Member, server xuất hiện trong sidebar, tôi được redirect đến kênh #general
AC3: Link đã hết hạn
GivenLink mời đã quá 7 ngày
WhenNgười dùng mở link
ThenHiển thị trang lỗi: "Link mời đã hết hạn hoặc không hợp lệ."
Priority: High Persona: Anh Đạt + Lan Anh Tech: Token-based invite
LB-203 Kick / Ban thành viên khỏi server
As a Server Owner (Anh Đạt),
I want to kick hoặc ban thành viên vi phạm quy tắc khỏi server,
So that tôi duy trì được môi trường giao tiếp lành mạnh và an toàn cho cộng đồng.
Acceptance Criteria (BDD)
AC1: Kick thành viên
GivenTôi là Owner, mở danh sách thành viên và click vào một Member
WhenTôi chọn "Kick" và xác nhận trong dialog
ThenThành viên bị xóa khỏi server ngay lập tức, server biến mất khỏi sidebar của họ, nhưng họ có thể join lại qua invite link mới
AC2: Ban thành viên (vĩnh viễn)
GivenTôi là Owner, mở danh sách thành viên và click vào một Member
WhenTôi chọn "Ban" và xác nhận
ThenThành viên bị xóa khỏi server và bị chặn vĩnh viễn – mọi invite link đều không cho phép họ join lại
AC3: Không thể kick/ban chính mình
GivenTôi là Owner và xem profile của chính mình trong member list
WhenTôi mở context menu
ThenCác nút "Kick" và "Ban" bị ẩn hoặc disabled
Priority: High Persona: Anh Đạt – Server Owner Tech: REST API + WebSocket broadcast
LB-204 Rời khỏi server (Leave Server)
As a thành viên (Minh Khoa),
I want to tự rời khỏi server mà tôi không còn muốn tham gia,
So that sidebar của tôi chỉ hiển thị các server tôi thực sự hoạt động.
Acceptance Criteria (BDD)
AC1: Rời server thành công
GivenTôi right-click vào icon server trong sidebar
WhenTôi chọn "Rời server" và xác nhận trong dialog
ThenTôi bị xóa khỏi danh sách thành viên, server biến mất khỏi sidebar, danh sách member cập nhật realtime cho mọi người
AC2: Owner không thể rời server
GivenTôi là Owner duy nhất của server
WhenTôi cố chọn "Rời server"
ThenHiển thị thông báo: "Bạn là Owner duy nhất. Hãy chuyển quyền Owner hoặc xóa server trước khi rời."
Priority: Medium Persona: Minh Khoa – Active Member Tech: REST API + WebSocket
💬 Epic 3 – Channel & Real-time Messaging
LB-301 Gửi và nhận tin nhắn thời gian thực
As a thành viên (Minh Khoa),
I want to gửi tin nhắn văn bản trong kênh chat và nhận tin nhắn từ người khác ngay lập tức,
So that cuộc trò chuyện diễn ra tự nhiên, không cần reload trang.
Acceptance Criteria (BDD)
AC1: Gửi tin nhắn thành công
GivenTôi đang ở kênh text và nhập tin nhắn vào ô chat
WhenTôi nhấn Enter hoặc nút Send
ThenTin nhắn xuất hiện ngay trong message list với timestamp; tất cả thành viên online trong kênh nhận được trong ≤ 500ms qua WebSocket
AC2: Nhận tin nhắn realtime
GivenTôi đang mở kênh chat và có thành viên khác gửi tin nhắn mới
WhenBackend broadcast qua WebSocket
ThenTin nhắn hiện ngay trong UI của tôi không cần bất kỳ thao tác nào
AC3: Lịch sử tin nhắn khi vào kênh
GivenTôi mở một kênh text bất kỳ
WhenTrang kênh load xong
Then50 tin nhắn gần nhất được hiển thị, cuộn lên để lazy-load thêm theo batch 50 tin
Priority: High Persona: Minh Khoa – Active Member Tech: WebSocket / STOMP
LB-302 Tạo và quản lý kênh text trong server
As a Server Owner (Anh Đạt),
I want to tạo, đổi tên và xóa kênh text trong server của mình,
So that tôi có thể tổ chức cuộc trò chuyện theo từng chủ đề rõ ràng.
Acceptance Criteria (BDD)
AC1: Tạo kênh text mới
GivenTôi là Owner, click "+ Thêm kênh" trong sidebar
WhenTôi đặt tên kênh (lowercase, không dấu, ≤ 50 ký tự) và chọn loại "Text"
ThenKênh được tạo, xuất hiện ngay trong danh sách kênh của tất cả thành viên server
AC2: Xóa kênh
GivenTôi là Owner, right-click vào kênh → "Xóa kênh"
WhenTôi xác nhận trong hộp thoại confirm
ThenKênh và toàn bộ lịch sử tin nhắn bị xóa, kênh biến mất khỏi sidebar của tất cả thành viên trong ≤ 1 giây
Priority: Medium Persona: Anh Đạt – Server Owner Tech: REST + WebSocket broadcast
LB-303 Xem danh sách thành viên và trạng thái online
As a người dùng mới (Lan Anh) hoặc thành viên (Minh Khoa),
I want to xem danh sách tất cả thành viên trong server và biết ai đang online,
So that tôi biết cộng đồng có active không và có thể tương tác với ai ngay lúc này.
Acceptance Criteria (BDD)
AC1: Hiển thị member list panel
GivenTôi đang ở bất kỳ kênh text nào trong server
WhenTôi click icon "Danh sách thành viên" ở toolbar hoặc panel tự mở mặc định
ThenPanel phải hiển thị hai nhóm: "Online" (dot xanh) và "Offline" (dot xám), sắp xếp theo display name A→Z
AC2: Cập nhật trạng thái realtime
GivenPanel member list đang mở
WhenMột thành viên đăng nhập hoặc đăng xuất
ThenThành viên đó di chuyển giữa nhóm Online/Offline trong ≤ 3 giây mà không cần reload
AC3: Hiển thị role Owner
GivenPanel member list đang mở
WhenServer có Owner và Member
ThenOwner được hiển thị với badge/icon đặc biệt (vương miện hoặc màu khác) để phân biệt
Priority: Medium Persona: Lan Anh + Minh Khoa Tech: WebSocket presence
🎙️ Epic 4 – Voice Channel
LB-401 Tham gia và rời voice channel
As a thành viên (Minh Khoa),
I want to tham gia voice channel trong server chỉ bằng một click và thấy ai đang trong phòng,
So that tôi có thể họp / trò chuyện trực tiếp ngay mà không cần app họp riêng.
Acceptance Criteria (BDD)
AC1: Tham gia voice channel
GivenTôi thấy kênh voice trong sidebar và đã cấp quyền microphone cho trình duyệt
WhenTôi click vào tên kênh voice
ThenTôi kết nối vào phòng voice, avatar của tôi xuất hiện dưới tên kênh, các thành viên khác nghe được tiếng tôi
AC2: Hiển thị người đang nói (speaking indicator)
GivenCó ≥ 2 người trong voice channel
WhenMột thành viên đang nói
ThenAvatar của người đó sáng lên / có viền xanh trong UI của tất cả thành viên
AC3: Rời voice channel
GivenTôi đang trong voice channel
WhenTôi nhấn nút "Ngắt kết nối" (icon điện thoại đỏ)
ThenKết nối voice bị đóng, avatar tôi biến mất khỏi danh sách phòng, micro không còn active
AC4: Trình duyệt từ chối quyền micro
GivenNgười dùng chặn quyền microphone trên trình duyệt
WhenTôi cố tham gia voice channel
ThenHiển thị toast error: "Cần quyền microphone để vào voice channel. Vui lòng kiểm tra cài đặt trình duyệt."
Priority: High Persona: Minh Khoa – Active Member Tech: WebRTC / SFU
🔔 Epic 5 – Notification & Presence
LB-501 Thông báo tin nhắn chưa đọc
As a thành viên (Minh Khoa),
I want to thấy badge số tin nhắn chưa đọc trên icon server và tên kênh,
So that tôi không bỏ lỡ tin nhắn quan trọng kể cả khi đang ở kênh khác.
Acceptance Criteria (BDD)
AC1: Badge unread trên kênh
GivenTôi đang ở kênh A và có tin nhắn mới đến kênh B
WhenWebSocket nhận broadcast tin nhắn mới tại kênh B
ThenTên kênh B in đậm và hiển thị badge số tin nhắn chưa đọc ở sidebar
AC2: Xóa unread khi vào kênh
GivenKênh B đang có 5 tin nhắn chưa đọc
WhenTôi click vào kênh B
ThenBadge biến mất, hệ thống đánh dấu tất cả tin nhắn là "đã đọc" trong DB
Priority: Medium Persona: Minh Khoa – Active Member Tech: WebSocket + PostgreSQL read receipts
👤 Epic 6 – User Profile
LB-601 Cập nhật hồ sơ cá nhân (avatar & display name)
As a thành viên (Minh Khoa),
I want to cập nhật ảnh đại diện và tên hiển thị của mình,
So that các thành viên khác dễ dàng nhận diện tôi trong server.
Acceptance Criteria (BDD)
AC1: Đổi display name
GivenTôi mở Settings → Profile
WhenTôi nhập tên mới (1–50 ký tự) và nhấn "Lưu"
ThenDisplay name cập nhật trên tất cả server tôi tham gia; các tin nhắn mới hiển thị tên mới, tin nhắn cũ giữ nguyên tên lúc gửi
AC2: Upload avatar
GivenTôi đang ở trang Profile
WhenTôi upload ảnh PNG/JPG ≤ 2MB
ThenẢnh hiển thị preview tròn; sau khi lưu, avatar mới xuất hiện ở header, member list, và bên cạnh tin nhắn của tôi
AC3: Validation tên trống
GivenTôi xóa hết ký tự trong field display name
WhenTôi nhấn "Lưu"
ThenHiển thị lỗi: "Tên hiển thị không được để trống." và không lưu thay đổi
Priority: Medium Persona: Minh Khoa – Active Member Tech: REST API + File Upload
