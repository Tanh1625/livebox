Project Description

Purpose

LiveBox được xây dựng nhằm mục đích học tập và thực hành công nghệ, đồng thời giải quyết bài toán thực tiễn: thiết kế một nền tảng chat thời gian thực hoàn chỉnh theo kiến trúc client–server hiện đại, tương tự Discord. Dự án cung cấp môi trường để nhóm sinh viên trải nghiệm toàn bộ vòng đời phát triển phần mềm – từ phân tích nghiệp vụ (BA), thiết kế hệ thống, lập trình, đến kiểm thử – dưới sự hướng dẫn của giảng viên.

Về phía người dùng, LiveBox cho phép tạo server riêng, tổ chức kênh chat theo chủ đề, nhắn tin thời gian thực và voice call ngay trong trình duyệt, mà không cần cài thêm ứng dụng bên ngoài (BG-03). Mục tiêu onboarding đặt ra là hoàn thành đăng ký trong ≤ 30 giây (BG-01) và độ trễ tin nhắn ≤ 500ms (BG-02).

Scope

Bao gồm (In-scope – Phase 1 MVP):

  • Xác thực người dùng: Đăng ký, Đăng nhập, Đăng xuất an toàn với JWT Access / Refresh Token (LB-101, LB-102, LB-103).
  • Quản lý Server: Tạo server, mời thành viên qua invite link có TTL 7 ngày, kick/ban thành viên, rời server (LB-201 → LB-204).
  • Messaging: Gửi/nhận tin nhắn realtime qua WebSocket/STOMP, quản lý kênh text, danh sách thành viên online (LB-301 → LB-303).
  • Voice Channel: Tham gia voice call 1-click qua WebRTC, tối đa 20 người/phòng, speaking indicator (LB-401).
  • Thông báo & Profile: Badge unread theo kênh, cập nhật avatar và display name (LB-501, LB-601).
  • Tài liệu BA: User Personas, User Stories (13 stories / 6 Epics), Business Goals, Stakeholder Matrix, Project Charter.

Không bao gồm (Out-of-scope – Phase 1):

  • Ứng dụng mobile native (iOS / Android).
  • Chia sẻ file / hình ảnh trong kênh chat.
  • Emoji reaction và thread / reply tin nhắn.
  • Hệ thống moderation tự động (AI spam filter).
  • Persistent voice recording / screen sharing.
  • Redis hoặc bất kỳ cache layer nào (Phase 1 dùng PostgreSQL trực tiếp).

Key Deliverables

1. Frontend Web App – ReactJS (TypeScript): giao diện server sidebar, kênh chat, voice channel UI, trang profile.
2. Backend REST API + WebSocket Server – Spring Boot + Spring Security + JWT + WebSocket/STOMP.
3. Database Schema – PostgreSQL: bảng users, servers, channels, messages, memberships, invite_codes.
4. Bộ tài liệu BA đầy đủ: User Personas v0.1, User Stories v0.1 (13 stories), Business Goals v0.1, Stakeholder Matrix v0.1, Project Charter v0.1.
5. Môi trường Demo – Deploy lên free-tier cloud (Render / Railway + Supabase), có URL public để demo với giảng viên.

Risks

Potential Risk Perceived Impact Proposed Mitigation Strategy
WebRTC không hoạt động trên môi trường free-tier (NAT traversal / TURN server) Cao (High) Tích hợp Cloudflare TURN miễn phí hoặc dùng Metered.ca TURN (free quota). Ưu tiên test voice từ Sprint 3, không để Sprint cuối mới phát hiện.
WebSocket bị timeout / disconnect trên Render free-tier (inactivity sleep) Cao (High) Dùng keep-alive ping hoặc chuyển sang Railway (ít sleep hơn). Thêm auto-reconnect logic ở phía client (React).
Scope creep – thêm tính năng ngoài Phase 1 (emoji, file sharing) trong quá trình code Trung bình (Medium) BA giữ vững Scope bằng cách đóng băng User Stories Phase 1. Tính năng mới ghi nhận vào Phase 2 Backlog, không implement ngay.
Vibe Coding tạo ra code quality thấp, khó maintain hoặc có lỗ hổng bảo mật JWT Trung bình (Medium) Review code cuối mỗi sprint với giảng viên. Dùng Spring Security defaults cho JWT – không tự implement crypto. Có unit test cho luồng Auth.
Thiếu nhân lực / thời gian 1 tháng không đủ hoàn thành 13 User Stories Trung bình (Medium) Ưu tiên Must-Have Stories trước (LB-101, LB-102, LB-201, LB-301, LB-401). Nếu trễ, cắt LB-601 (Profile) và LB-204 (Leave) sang Phase 2.
PostgreSQL trên Supabase free-tier bị suspend do inactivity Thấp (Low) Dùng Neon.tech thay thế (không tự suspend). Script seed data sẵn để restore nhanh khi cần demo.
