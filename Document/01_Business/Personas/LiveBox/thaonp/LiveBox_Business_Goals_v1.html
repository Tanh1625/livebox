Business Goals & System KPIs

As-Is Constraints & To-Be Objectives for the LiveBox Real-time Chat Platform
Version: v1.0 (Baseline)

1. As-Is Context & Current Pain Points

Derived from the project requirements and persona analysis of Admins, Authenticated Members, and External Visitors, the current communication methods suffer from three major bottlenecks:

  • High Latency & Resource Drain: Legacy HTTP polling mechanisms generate massive network overhead, causing message delivery delays, UI freezes, and excessive server load.
  • Disjointed UX & State Desynchronization: Creating structural entities like Servers or Channels requires tedious form submissions and hard page reloads (F5) to update the view hierarchy.
  • Security Vulnerabilities: Lack of granular Role-Based Access Control (RBAC) and fragile session persistence, leading to poor onboarding and unhandled 401 Unauthorized exceptions for API endpoints.

2. SMART Business & Technical Goals

The LiveBox platform aims to resolve these issues by implementing a modern Client-Server architecture. The system must hit the following targets corresponding with the implementation phase:

1. Ultra-Low Latency Messaging

< 200ms Latency

Ensuring real-time conversational flows without breaking the immersion constraints of end users.

  • Target: Maintain end-to-end text message broadcasting latency below 200ms across active subscribers.
  • LiveBox Solution: Replacing HTTP polling with persistent WebSocket connections via Spring Boot message brokers. The ReactJS SPA will utilize Optimistic UI patterns to instantly render pending state mutations while awaiting network ACKs.

2. Seamless Workspace Provisioning

Zero Page Reloads (SPA)

Providing Community Admins with dynamic and intuitive structure management.

  • Target: 100% elimination of full page refreshes when provisioning Servers or Text/Voice Channels.
  • LiveBox Solution: Mapping RESTful DTOs directly to PostgreSQL entities for persistence. The React (TypeScript) frontend will intercept these state mutations to instantly re-render the Sidebar component tree, ensuring seamless context switching.

3. Scalable Security & Data Integrity

Robust Authorization Pipeline

Securing the concurrent architecture to handle thousands of isolated tenant sessions securely without cross-contamination.

  • Target: Guarantee zero unauthorized access to Private API routes and ensure isolated database transactions with complete ACID compliance during message insertions.
  • LiveBox Solution: Implementing Spring Security to issue stateless JWTs encapsulated inside secure HttpOnly cookies. API Gateways will gracefully intercept 401 statuses to redirect visitors, while PostgreSQL acts as the single source of truth for scalable Data Integrity.
